# Security Policy

## Supported Versions

The following versions of this project are currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you have discovered a security vulnerability, we appreciate your help in disclosing it into a responsible manner.

### How to report
Please DO NOT report security vulnerabilities through public GitHub issues.

Instead, please report them via the "Private vulnerability reporting" feature in the Security tab of this repository, or email the project maintainer directly.

### What to include
Please provide as much detail as possible, including:
1.  The type of issue (e.g., buffer overflow, SQL injection, XSS).
2.  Full paths of source file(s) related to the manifestation of the issue.
3.   The location of the affected source code (tag/branch/commit or direct URL).
4.  Any special configuration required to reproduce the issue.
5.  Step-by-step instructions to reproduce the issue.
6.  Proof-of-Concept or Exploit code (if possible).
7.  Impact of the issue, including how an attacker might exploit the issue.

### Timeline
We will acknowledge your report within 48 hours and aim to resolve all critical vulnerabilities within 7 days.
